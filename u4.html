Proyecto Final

TRABAJO FINAL SOBRE E-COMERCE

Contexto

    Imagina que entras a la página y te logueas. Una vez adentro, buscas y seleccionas esos zapatos geniales que quieres. Los añades al carrito y decides comprarlos. Entonces, ingresas los datos de pago y la página se encarga de verificar que todo esté bien en la base de datos, confirmando tu compra. Aquí viene lo bueno: después de confirmar, la base de datos actualiza el inventario, quitando esos zapatos de la lista disponible. ¡Y listo! Has completado el proceso de compra, y esos zapatos que elegiste ya no están disponibles para otros compradores.


    Usuario
  1. Inicio del Proceso: El usuario inicia sesión en la página web.
  2. Selección de Producto: El usuario navega por el catálogo de zapatos y selecciona un par para comprar.
  3. Añadir al Carrito: El usuario agrega el par de zapatos seleccionado al carrito de compras.
  4. Proceso de Pago: El usuario procede a realizar el pago.
  5. Confirmación: La página web verifica el pago. La página web actualiza la base de datos con la compra realizada.
  6. Fin del Proceso: El usuario recibe una confirmación de la compra.

    7. Pagina web
  7. Inicio del Proceso: Interfaz para iniciar sesión y navegación por los productos.
  8. Selección de Producto: Presentación de catálogo y opciones para seleccionar zapatos.
  9. Añadir al Carrito: Interacción para agregar el producto seleccionado al carrito.
  10. Confirmación: Verificación del pago y actualización de la base de datos.
  11. Fin del Proceso: Mensaje de confirmación de la compra al usuario.

    12. Base de datos
  12. Inicio del Proceso: Registra la información de inicio de sesión del usuario.
  13. Recepción de Pedido: Guarda los detalles del pedido en la base de datos.
  14. Actualización de Stock: Reduce la cantidad de zapatos disponibles en el inventario.
  15. Fin del Proceso: Confirma la transacción y el estado actualizado del inventario.

    16. MODELO BPMN

    Requisitos Funcionales
  16. Inicio del Proceso (Usuario): La página web debe proporcionar una interfaz segura y fácil para que los usuarios inicien sesión.
  17. Selección de Producto (Usuario): La página web debe presentar un catálogo de zapatos de manera clara y organizada.
  18. Añadir al Carrito (Usuario): Debe haber una funcionalidad que permita a los usuarios agregar productos al carrito de compras.

    19. Requisitos de Usabilidad
  19. Inicio del Proceso (Usuario): La interfaz de inicio de sesión debe ser intuitiva y fácil de usar.
  20. Selección de Producto (Usuario): La presentación del catálogo debe ser atractiva y fácil de navegar.
  21. Añadir al Carrito (Usuario): La interacción para agregar productos al carrito debe ser clara y sin complicaciones.
    22. Requisitos de Seguridad
  22. Inicio del Proceso (Usuario): Se deben implementar medidas de seguridad para proteger la información de inicio de sesión.
  23. Proceso de Pago (Usuario): La página web debe garantizar la seguridad de los datos de pago durante la transmisión.
    24. Requisitos de Rendimiento
  24. Inicio del Proceso (Página Web): La página web debe cargar rápidamente para proporcionar una experiencia de usuario fluida.
  25. Proceso de Pago (Página Web): La verificación del pago debe realizarse de manera eficiente y sin demoras.
    26. Requisitos de Integración
  26. Actualización de Stock (Base de Datos): La base de datos debe integrarse de manera efectiva con la página web para actualizar el inventario después de cada compra.
    27. Requisitos de Mantenimiento
  27. Inicio del Proceso (Base de Datos): El sistema debe ser capaz de manejar y mantener registros de información de inicio de sesión de usuarios.
  28. Fin del Proceso (Base de Datos): La base de datos debe ser fácil de mantener y permitir actualizaciones de inventario de manera eficiente.
    29. Requisitos de Fiabilidad:
  29. Proceso de Pago (Página Web): La página web debe ser confiable al verificar y procesar los pagos de manera precisa.
