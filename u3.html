UNIDAD_3 - FUNDAMENTOS DE INGENERIA DE SOFTWARE

Investigación

Conceptos

  1. Modelos de madures

    2. CMMI (Capability Maturity Model Integration): Es un modelo de madurez que proporciona una guía para la mejora continua de los procesos y la capacidad organizativa. CMMI abarca áreas clave como desarrollo, servicios, adquisición y gestión.

    PSP (Personal Software Process) y TSP (Team Software Process): Son modelos de madurez desarrollados por el Software Engineering Institute (SEI). El PSP se centra en mejorar las habilidades individuales del desarrollador, mientras que el TSP se enfoca en mejorar la eficiencia y calidad del equipo de desarrollo.

  2. Estándares e Métricas de Calidad

    3. ISO 25000 (SQuaRE - Software Quality Requirements and Evaluation): Define estándares para la calidad del software y sistemas. Proporciona un marco para la evaluación y mejora de la calidad del software.

  3. Enfoque de Proceso

    4. MOPROSOFT (Modelo de Procesos para la Industria del Software): Es un modelo de procesos de calidad desarrollado en México que se utiliza para evaluar y mejorar la gestión de procesos en la industria del software.

  4. SPACE, SP y ISE

    5. SPACE (Software Process Assessment and Capability Determination): Es un modelo de evaluación de procesos de software utilizado para determinar la capacidad y madurez de los procesos de una organización.

    SP (Software Process): Es un modelo de procesos de software que aborda la planificación y ejecución de los procesos, con un enfoque en la mejora continua.

    ISE (Integrated Systems Engineering): Se refiere a la ingeniería integrada de sistemas y se centra en la gestión de sistemas complejos a lo largo de su ciclo de vida.
